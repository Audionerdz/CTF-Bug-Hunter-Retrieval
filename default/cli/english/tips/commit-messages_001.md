## Tip: Effective Commit Messages

```bash
# Bad - not descriptive
git commit -m "update"

# Good - clear and specific
git commit -m "fix: Correct typo in README.md"

# Better - with context
git commit -m "feat: Add SQL injection guide with 5 payloads

- Includes bypass techniques
- Real lab examples
- Remediation section"
```
